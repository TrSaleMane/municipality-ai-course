# 🔒 自治体AIシステム セキュリティ・要件チェックリスト
## 新人SE向け 自治体AIシステム開発 模擬体験講座 Day3

> 自治体システムには一般のWebサービスとは異なる厳格な要件があります。
> 本番システムを開発する際の確認リストとして活用してください。

---

## 1. 個人情報保護

### 1-1. 個人情報の取り扱い

- [ ] **収集の最小化**：システムが不必要な個人情報を収集しない
- [ ] **目的の明示**：収集する情報の利用目的を明示している
- [ ] **氏名・住所の非収集**：チャットボットが氏名・住所を聞かない設計になっている
- [ ] **マイナンバーの非収集**：マイナンバーをシステム内で扱わない
- [ ] **ログの管理**：会話ログを適切に管理・保護している
- [ ] **保存期間の定義**：データの保存期間と削除ルールが定められている

### 1-2. 法令対応

- [ ] **個人情報保護法**：個人情報保護法の要件を確認している
- [ ] **マイナンバー法**：特定個人情報（マイナンバー）の取り扱い規制を確認している
- [ ] **条例対応**：各自治体の個人情報保護条例を確認している
- [ ] **プライバシーポリシー**：利用者向けのプライバシーポリシーを作成している

---

## 2. セキュリティ

### 2-1. アクセス制御

- [ ] **認証の実装**：利用者認証（ログイン）の仕組みがある
- [ ] **権限管理**：職員・住民・管理者など役割ごとにアクセス権限が異なる
- [ ] **セッション管理**：セッションタイムアウトが設定されている
- [ ] **IPアドレス制限**：必要に応じてアクセス元を制限している（職員向けシステム）

### 2-2. 通信・データ保護

- [ ] **HTTPS**：すべての通信がHTTPS（TLS1.2以上）で暗号化されている
- [ ] **APIキーの保護**：LLMのAPIキーがソースコードにハードコードされていない
- [ ] **環境変数の利用**：機密情報は環境変数またはシークレット管理サービスで管理
- [ ] **保存データの暗号化**：DBや保存ファイルが暗号化されている

### 2-3. 脆弱性対策

- [ ] **プロンプトインジェクション対策**：悪意ある入力でシステムが誤動作しない
- [ ] **入力値検証**：ユーザー入力の文字数・内容の検証がある
- [ ] **レート制限**：大量リクエスト（DoS）に対する制限がある
- [ ] **依存ライブラリの管理**：使用ライブラリの脆弱性を定期確認している

### 2-4. ログ・監査

- [ ] **アクセスログ**：誰がいつシステムを利用したか記録している
- [ ] **エラーログ**：システムエラーを適切に記録・通知している
- [ ] **操作ログ**：重要操作（管理者機能）のログを取得している
- [ ] **ログの保護**：ログ自体の改ざん防止対策がある

---

## 3. AIシステム特有のリスク

### 3-1. ハルシネーション（誤情報）対策

- [ ] **RAGの活用**：根拠となる文書に基づいて回答する仕組みになっている
- [ ] **回答根拠の表示**：回答の情報源を明示できる
- [ ] **不確実性の表明**：わからないことを「わからない」と答える
- [ ] **免責事項**：AIの回答は参考情報であり、正式な情報は窓口で確認するよう案内している
- [ ] **定期的な精度確認**：定期的に回答精度を評価・改善している

### 3-2. バイアス・公平性

- [ ] **差別的回答の防止**：属性（年齢・性別・国籍等）による差別的対応をしない
- [ ] **多言語対応の検討**：外国語話者への対応を検討している
- [ ] **デジタルデバイドへの配慮**：AIが使えない方向けの代替手段がある

### 3-3. 透明性・説明責任

- [ ] **AI使用の明示**：利用者にAIであることを明示している
- [ ] **人間への引き継ぎ**：複雑な案件は担当者に引き継げる仕組みがある
- [ ] **フィードバック機能**：回答の評価・フィードバックを収集できる
- [ ] **改善プロセス**：フィードバックをもとにシステムを改善するプロセスがある

---

## 4. アクセシビリティ

### 4-1. JIS X 8341-3（Webアクセシビリティ）

- [ ] **文字サイズ**：文字が十分な大きさで表示される（最低14px以上）
- [ ] **色コントラスト**：文字と背景のコントラスト比が4.5:1以上
- [ ] **キーボード操作**：マウスなしでも操作できる
- [ ] **スクリーンリーダー対応**：読み上げソフトで内容が伝わる
- [ ] **エラーメッセージ**：エラー内容が具体的にわかる

### 4-2. ユニバーサルデザイン

- [ ] **わかりやすい言葉**：難しい行政用語を避けた平易な表現
- [ ] **ふりがな**：必要に応じてふりがなを表示できる
- [ ] **高齢者への配慮**：シンプルな操作で利用できる
- [ ] **スマートフォン対応**：スマートフォンでも使いやすいUI

---

## 5. 運用・保守

### 5-1. システム可用性

- [ ] **稼働時間の定義**：システムの稼働時間（24時間/平日のみ 等）が定義されている
- [ ] **障害時の対応**：システム停止時の代替手段が用意されている
- [ ] **バックアップ**：データのバックアップが定期的に取得されている
- [ ] **復旧手順**：障害発生時の復旧手順が文書化されている

### 5-2. ナレッジ管理

- [ ] **FAQの更新プロセス**：法改正・制度変更に合わせてFAQを更新するプロセスがある
- [ ] **更新担当者の定義**：誰がFAQを更新するか決まっている
- [ ] **バージョン管理**：FAQデータのバージョン管理をしている
- [ ] **定期レビュー**：定期的にFAQの内容を見直している
