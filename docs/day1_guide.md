## 1. Google Colab の基本的な使い方

### Colab とは

Google Colaboratory（略称：Colab）は、ブラウザ上で Python を実行できる無料の開発環境です。  
インストール不要で、Google アカウントがあればすぐに使えます。

| 項目 | 内容 |
|------|------|
| 必要なもの | Google アカウント、ブラウザ |
| 費用 | 無料（GPU/TPU の大量利用は有料プランあり） |
| 保存先 | Google ドライブ |
| 実行環境 | Linux（Ubuntu）上の Python |

### ノートブックを開く

**方法 A：「Open in Colab」バッジをクリック**

`day1_llm_basic.ipynb` の冒頭にある下記バッジをクリックします。

[![Open In Colab](https://colab.research.google.com/assets/colab-badge.svg)](https://colab.research.google.com/github/TrSaleMane/municipality-ai-course/blob/main/docs/day1_llm_basic.ipynb)

**方法 B：Colab から直接開く**

1. [https://colab.research.google.com](https://colab.research.google.com) にアクセス
2. 「ファイル」→「GitHub からノートブックを開く」
3. URL に `https://github.com/TrSaleMane/municipality-ai-course` を入力
4. `docs/day1_llm_basic.ipynb` を選択

### 自分の Google ドライブにコピーする（推奨）

他の人の変更に影響されず、自分の作業内容を保存するために**必ずコピーを作成**してください。

1. メニューバーから「ファイル」→「ドライブにコピーを保存」をクリック
2. 「〇〇のコピー」という名前で自分のドライブに保存されます
3. 以降はそのコピーを使って演習を進めます

### 画面の見方

```
┌─────────────────────────────────────────────┐
│  [ファイル][編集][表示][挿入][ランタイム][ツール]  ← メニューバー
├──────┬──────────────────────────────────────┤
│      │  [+ コード] [+ テキスト]              ← セル追加ボタン
│ サイド│                                       │
│ バー  │  ▶ [ コードセル ]                    ← 実行ボタン
│（鍵等）│  ─────────────────────────────────  │
│      │  [ テキストセル（説明文） ]             │
└──────┴──────────────────────────────────────┘
```

### ランタイムへの接続

ノートブックを開いたら、最初に**ランタイム（実行環境）に接続**する必要があります。

1. 右上の「接続」ボタンをクリック（または任意のセルを実行すると自動接続）
2. 「RAM ●GB / ディスク ●GB」と表示されたら接続完了

> ランタイムは**90分間操作がないと切断**されます。  
> 切断された場合は再接続して、セルを最初から順番に再実行してください。

### セルの実行方法

| 操作 | 内容 |
|------|------|
| `Shift + Enter` | 現在のセルを実行し、次のセルへ移動 |
| `Ctrl + Enter` | 現在のセルを実行し、同じセルにとどまる |
| ▶ ボタンをクリック | セル左端の再生ボタンをクリックして実行 |
| `Ctrl + F9` | ノートブック全体を最初から実行 |

実行中はセル左端に `[*]` が表示されます。  
実行が完了すると `[番号]`（例：`[1]`）に変わります。

## 2. ノートブック（.ipynb）の進め方

### セルの種類

`.ipynb` ファイルは「セル」の集まりでできています。

| セルの種類 | 見た目 | 役割 |
|-----------|--------|------|
| **コードセル** | グレー背景・左端に `[ ]` | Python コードを記述・実行する |
| **テキストセル** | 白背景・Markdown 形式 | 説明・図解・注意事項を表示する |

### 進め方のルール

**1. 上から順番に実行する**  
セル同士は変数や関数を共有しています。順番を飛ばすとエラーになります。

```
Section 1 → Section 2 → Section 3 → ... の順に進める
```

**2. テキストセルを読んでから実行する**  
各セクションの冒頭にある説明（テキストセル）を読んでから  
コードセルを実行することで、何をしているのかを理解しながら進められます。

**3. コードを自由に書き換えてみる**  
コードセルのテキストは自由に編集できます。  
パラメータを変えたり、プロンプトを書き換えたりして、挙動の変化を確認しましょう。

**4. チャレンジ課題に取り組む**  
ノートブック末尾の「チャレンジ課題」は必須ではありませんが、  
理解を深めるための良い機会です。時間が余ったら挑戦してみてください。

### ノートブックの構成（Day 1）

| セクション | 内容 |
|-----------|------|
| Section 1 | 環境セットアップ（ライブラリのインストール・APIキー設定） |
| Section 2 | LLM とは何か / Gemini API で最初のチャット |
| Section 3 | プロンプトエンジニアリング入門（良いプロンプト vs 悪いプロンプト） |
| Section 4 | チャット履歴の管理（会話の流れを保持する） |
| Section 5 | インタラクティブチャット体験 |
| チャレンジ | システムプロンプト改善・別業務対応・フィードバック機能 |

## 3. APIキーの取得と設定

### Gemini API キーの取得

1. [Google AI Studio](https://aistudio.google.com/apikey) にアクセス
2. 「APIキーを作成」をクリック
3. 表示されたキー（`AIza...` で始まる文字列）をコピー

> 無料枠：1分あたり60リクエスト、1日あたり1,500リクエスト（2025年現在）

### Colab シークレットへの登録（推奨）

APIキーをコードに直接貼り付けるのは**セキュリティ上危険**です。  
Colab の「シークレット」機能を使って安全に管理しましょう。

1. 左サイドバーの **鍵アイコン** をクリック
2. 「+ 新しいシークレットを追加」をクリック
3. 名前に `GEMINI_API_KEY`、値に取得したAPIキーを入力
4. 「ノートブックのアクセス」トグルをオンにする

コード内での読み込み：

```python
from google.colab import userdata
GEMINI_API_KEY = userdata.get('GEMINI_API_KEY')
```

## 5. つまずきやすいポイントと対処法

### エラー：`ModuleNotFoundError: No module named 'google.generativeai'`

**原因：** Section 1 のインストールセルが実行されていない  
**対処：** Section 1 のセルを実行してから再試行

### エラー：`KeyError: 'GEMINI_API_KEY'`

**原因：** シークレットが登録されていない、またはノートブックへのアクセスが無効  
**対処：** シークレット設定を確認し、「ノートブックのアクセス」トグルがオンになっているか確認

### エラー：`PermissionDenied: 403`

**原因：** APIキーが間違っているか、APIが有効化されていない  
**対処：** [Google AI Studio](https://aistudio.google.com/apikey) でキーを再確認

### ランタイムが切断された

**対処：**
1. 右上の「再接続」をクリック
2. メニューから「ランタイム」→「前に実行したセルをすべて実行」を選択

### `while True` のセル（インタラクティブチャット）が止まらない

**対処：** セル左端の **■（停止ボタン）** をクリック、または `Ctrl + M + I`
