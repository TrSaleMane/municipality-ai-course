# 新人SE向け：自治体AIシステム開発 模擬体験講座

> ガバメントAI「源内」OSSをベースに、AIを活用した住民窓口チャットボットを開発する3日間の実践講座

## 講座概要

| 項目 | 内容 |
|------|------|
| 対象 | 新人SE（プログラミング基礎知識あり） |
| 期間 | 3日間 |
| 形式 | 講義 + ハンズオン + チーム演習 |
| 使用言語 | Python（メイン）、TypeScript（読解のみ） |
| 開発環境 | Google Colab、ローカルPC |
| 演習ベース | [ガバメントAI「源内」OSS](https://github.com/digital-go-jp/genai-web)（MIT License） |

---

## 学習目標

1. 実際の政府OSSを読んでアーキテクチャを理解できる
2. RAG（Retrieval-Augmented Generation）の仕組みを手を動かして体験できる
3. 自治体システム特有の要件（セキュリティ・アクセシビリティ・個人情報保護）を理解できる
4. AIシステム開発の全工程（要件定義 → 設計 → 実装 → 発表）を体験できる

---

## 自治体システムの主な種類

本講座では以下のような自治体業務システムを対象として扱います。

| カテゴリ | 代表的なシステム例 |
|----------|------------------|
| 住民サービス系 | 住民票・戸籍管理、転入出届、各種申請（健康保険・介護・障害福祉） |
| 税務・財務系 | 固定資産税・住民税の課税管理、予算・会計システム |
| 福祉・保健系 | 生活保護管理、児童手当、健康診断・予防接種管理 |
| 都市・土木系 | 道路台帳・橋梁管理、建築確認申請、開発許可、上下水道管理 |
| 教育・文化系 | 学校管理、就学支援、図書館システム |
| 内部業務系 | 人事・給与、文書管理、議会システム |

## 技術スタック

### 本番環境（源内OSS）

| コンポーネント | 使用技術 |
|--------------|---------|
| LLM基盤 | Amazon Bedrock |
| ベクトルDB | Amazon OpenSearch |
| 認証 | Amazon Cognito |
| インフラ管理 | AWS CDK |
| フロントエンド | React + TypeScript + Tailwind CSS |

### 講座用代替環境（Colab対応）

| 本番構成 | 講座用代替 | 理由 |
|---------|-----------|------|
| Amazon Bedrock | OpenAI API / Gemini API（無料枠） | コスト・手軽さ |
| Amazon OpenSearch | ChromaDB（ローカル or Colab） | セットアップ不要 |
| Amazon Cognito | 省略 or 簡易認証 | 学習スコープ外 |
| AWS CDK | Python + FastAPI | Python統一 |
| React フロントエンド | Gradio or Streamlit | 素早くUI構築可能 |

## カリキュラム

---

### Day 1：自治体システムとAIの概要理解

**テーマ：「なぜ自治体にAIが必要か」を知り、源内OSSの構造を読む**

#### 午前：自治体システムの全体像

- 自治体業務の種類と特徴（住民サービス・土木・福祉など）
- 従来のシステム開発と、AI導入後の変化
- ガバメントAI「源内」の誕生背景と目的
- デモ：源内の画面・機能を触ってみる

#### 午後前半：源内OSSのアーキテクチャ読解

- GitHubリポジトリの構成を確認する
- RAG（検索拡張生成）とは何か
- LLMとベクトルDBがどう連携しているか
- AWSサービス（Bedrock・OpenSearch・Cognito）の役割

#### 午後後半：Google ColabでLLM API呼び出し基礎体験

- Google ColabでPython環境をセットアップ
- OpenAI API / Gemini APIを使ってチャット応答を実装
- プロンプトエンジニアリング入門（システムプロンプトの書き方）

**Day 1 成果物：** LLMとチャットできるColabノートブック

---

### Day 2：RAG実装と住民窓口チャットボット構築

**テーマ：「住民FAQ文書を読んで答えるAI」を自分で作る**

#### 午前：ローカル + Colabで動く簡易版環境を構築

- ChromaDB（ローカルベクトルDB）のセットアップ
- 源内OSSのRAGアーキテクチャを参考にした設計
- 演習データ（自治体公開PDF・FAQ）の準備

#### 午後前半：ナレッジベース化とRAGチェーン実装

- PDF → テキスト抽出 → チャンク分割 → ベクトル化
- ChromaDBへの登録と類似検索
- RAGチェーンの実装（検索 → プロンプト生成 → LLM回答）

#### 午後後半：GradioでUIを作成して動かす

- Gradioで簡易チャットUIを構築
- 「住民窓口AI」として質問応答を体験
- 回答精度の確認・プロンプト改善

**Day 2 成果物：** 住民FAQ RAGチャットボット（Colab + Gradio）

---

### Day 3：カスタマイズ・セキュリティ・発表

**テーマ：「自分たちの業務テーマ」でオリジナル機能を追加し、発表する**

#### 午前：担当業務テーマ別カスタマイズ演習

チームで以下のいずれかのテーマを選んでオリジナル機能を追加する：

| テーマ | 演習内容 |
|--------|---------|
| 🏠 住民窓口 | 転入届・各種申請の手続き案内AIを強化 |
| 🛣️ 土木・インフラ | 道路損傷報告の受付・分類AI |
| 👨‍👩‍👧 福祉・子育て | 保育所申込・手当申請の質問応答AI |
| 🏫 教育 | 就学案内・学校行事のQ&A AI |

#### 午後前半：自治体システム特有の要件を学ぶ

- 個人情報保護法・マイナンバー法の基礎
- セキュリティ要件（ログ管理・アクセス制御・脆弱性対応）
- アクセシビリティ要件（JIS X 8341-3）
- AIシステム特有のリスク（幻覚・バイアス・情報漏洩）

#### 午後後半：チームデモ発表

- 5〜10分でシステムのデモと設計の説明
- 「なぜこの設計にしたか」を言語化する
- フィードバックセッション

**Day 3 成果物：** オリジナルテーマの自治体AIシステム + 発表資料
